final关键字的作用

final是Java中的一个修饰符，表示“不可变”、“最终”的意思，可用于变量、方法、类上。

一、用于变量：定义常量

✅ 意义：

  • 表示该变量是一个常量，值不能被修改
  • 如果变量是成员变量，建议配合static使用，写成：public static final，变成全局变量。

注意：

  • 基本类型：值不能变
  • 引用类型：引用不能变，但引用的对象的内容可以改变

二、用于方法：防止子类重写该方法

✅ 意义：

  • 表示这个方法是最终实现，子类无法对其进行重写（Override）
  • 常用于安全、稳定的基类设计中，防止被篡改逻辑

三、用于类：表示该类不可被继承

  • 表示该类是”最终类“，不能被继承
  • 常用语工具类、保证安全的类，如String、Math、Date、System

四、补充：final 在内部类、方法参数中也能用！

1.用于方法参数：防止方法中修改参数值

2.用于匿名内部类：访问外部变量时，该变量必须是final或effectively final（Java 8+）

在底层Lambda并不是直接访问原始变量，而是拷贝了一份值（final 或 effectively final）放到了Lambda的常量池/堆中。
在Java中局部变量是存储在栈帧中的（随着方法的调用而创建、销毁）；
而匿名内部类和Lambda实例是存储在堆中的（生命周期更长）。
所以为什么只能捕获final或effectively final？
因为Lambda要“复制值”，就必须保证这个值在整个过程中没有变动，否则拷贝的值就不能确定，运行逻辑会混乱。
无论是同步模式还是异步模式，在Lambda表达式中使用的变量都必须是final或effectively final。

⚠️PS: 常量必须初始化，且初始化后不能再被修改
